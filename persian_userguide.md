# RelayVPN — راهنمای کاربری

یک پراکسی محلی رایگان به‌صورت تک‌فایلی که ترافیک مرورگر شما را از طریق یک **رله Google Apps Script** عبور می‌دهد و مقصد واقعی را پشت ترافیک معمولی گوگل پنهان می‌کند. نه به VPS نیاز دارد، نه سرویس پولی — فقط یک حساب رایگان گوگل کافی است.

## این ابزار چه کاری انجام می‌دهد

شما یک برنامه کوچک را روی کامپیوتر خود اجرا می‌کنید. این برنامه روی `127.0.0.1:8085` (پراکسی HTTP) و `127.0.0.1:1080` (پراکسی SOCKS5) گوش می‌دهد. مرورگر خود را به آن وصل می‌کنید، و هر درخواستی که می‌فرستید از طریق یک Apps Script که خودتان روی حساب رایگان گوگلتان مستقر کرده‌اید، رله می‌شود.

از دید هر کسی که ترافیک بین کامپیوتر شما و اینترنت را شنود می‌کند، ترافیک شما شبیه HTTPS معمولی به `www.google.com` به نظر می‌رسد. وب‌سایتی که در واقع درخواست کرده‌اید توسط Apps Script شما واکشی می‌شود و پاسخ از همان کانال پنهان‌شده برمی‌گردد.

موارد استفاده این ابزار: یادگیری نحوه عملکرد domain fronting، آشنایی با مکانیزم TLS interception و پراکسی، و استفاده از مرورگر مقاوم در برابر سانسور در مکان‌هایی که دامنه‌های گوگل همچنان قابل دسترسی هستند.

## چگونه کار می‌کند

<div dir="ltr">

```
┌────────────┐       ┌──────────────┐       ┌──────────────────┐       ┌──────────┐
│  Browser   │──────▶│ relayvpn     │──────▶│ Google frontend  │──────▶│  Apps    │──▶ origin website
│ (your PC)  │◀──────│ (this tool)  │◀──────│ (TLS to google.) │◀──────│  Script  │◀── response
└────────────┘       └──────────────┘       └──────────────────┘       └──────────┘
                       │  MITM TLS         SNI = www.google.com         your free
                       │  using local CA   Host: script.google.com      Google account
                       │  HMAC-signed
                       │  envelope
```

</div>

سه نکته که باید در ذهن داشته باشید:

- پراکسی، ترافیک HTTPS شما را به‌صورت محلی رمزگشایی می‌کند و دوباره برای رله رمزگذاری می‌کند. به همین دلیل باید گواهی CA محلی آن را به‌عنوان یک گواهی معتبر نصب کنید — بدون این کار مرورگر شما (به‌درستی) از بارگذاری هر سایتی امتناع می‌کند.
- هر درخواست به رله با یک کلید مشترک که خودتان تعیین می‌کنید، امضا می‌شود. هیچ‌کس دیگری نمی‌تواند از Apps Script شما استفاده کند، حتی اگر در دسترس عموم باشد.
- رله Apps Script یک سهمیه دارد: حدود ۲۰٬۰۰۰ درخواست در روز برای هر حساب گوگل. اگر به این حد رسیدید، رله‌های بیشتری زیر حساب‌های گوگل دیگر مستقر کنید و همه را در `script_ids` لیست کنید.

## راه‌اندازی، گام به گام

### گام ۱ — ساخت فایل اجرایی

مخزن را کلون کنید و بسازید:

<div dir="ltr">

```bash
git clone https://github.com/SobhanYasami/GoMasterHttpRelayVPN.git
cd GoMasterHttpRelayVPN
make build
```

</div>

اکنون باید یک فایل اجرایی با نام `relayvpn` (یا `relayvpn.exe` در ویندوز) در ریشه پروژه داشته باشید.

اگر `make` در دسترس نیست، دستور زیر را مستقیماً اجرا کنید:

<div dir="ltr">

```bash
go build -trimpath -ldflags '-s -w' -o relayvpn ./cmd/relayvpn
```

</div>

### گام ۲ — تولید یک کلید مشترک قوی

این رمز عبوری است که پراکسی و رله از آن برای شناسایی یکدیگر استفاده می‌کنند. یک مقدار رمزنگاری‌شده تصادفی تولید کنید:

<div dir="ltr">

```bash
# Linux / macOS
openssl rand -base64 32
```

```powershell
# Windows PowerShell
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Minimum 0 -Maximum 256 }))
```

</div>

خروجی را کپی کنید. این مقدار را در دو جا قرار خواهید داد — یک بار در Apps Script و یک بار در فایل پیکربندی. **این دو باید دقیقاً یکسان باشند.**

یک کلید خوب اینگونه به نظر می‌رسد:

<div dir="ltr">

```
hv4Tx9YqM2sB1Kf7Lg8Qe6rZpWnH3uA5cV0jX8oI=
```

</div>

از رمز عبور تکراری از جای دیگر استفاده نکنید، و این رشته را در git قرار ندهید.

### گام ۳ — استقرار رله Apps Script

این بخش بیشترین کلیک را دارد، اما فقط یک‌بار انجامش می‌دهید.

۱. به <https://script.google.com> بروید و با حساب گوگل خود وارد شوید.
۲. روی **New project** (بالا-چپ) کلیک کنید.
۳. تمام کد نمونه در ویرایشگر را پاک کنید.
۴. فایل `apps_script/Code.gs` را از این مخزن باز کنید، تمام محتویاتش را کپی کنید، و در ویرایشگر پیست کنید.
۵. این خط را نزدیک بالای فایل پیدا کنید:

   <div dir="ltr">

   ```javascript
   const SHARED_SECRET = "CHANGE_ME_TO_A_STRONG_SECRET";
   ```

   </div>

   مقدار `CHANGE_ME_TO_A_STRONG_SECRET` را با کلیدی که در گام ۲ ساختید جایگزین کنید. علامت‌های نقل قول دور آن را حفظ کنید.
۶. روی نماد **Save** (دیسکت) کلیک کنید.
۷. روی **Deploy** → **New deployment** (بالا-راست) کلیک کنید.
۸. روی نماد چرخ‌دنده کنار «Select type» کلیک کرده و **Web app** را انتخاب کنید.
۹. فرم را پر کنید:
   - **Description:** هر چیز قابل به‌خاطر سپردن، مثلاً `relay v1`.
   - **Execute as:** `Me (your-email@gmail.com)`.
   - **Who has access:** `Anyone`.
۱۰. روی **Deploy** کلیک کنید. ممکن است گوگل از شما اجازه دسترسی بخواهد — تاییدها را بپذیرید. (هشدار «Google hasn't verified this app» را خواهید دید چون این برنامه شخصی شماست؛ روی **Advanced** → **Go to (project name)** کلیک کنید.)
۱۱. پس از استقرار، یک **Deployment ID** خواهید دید. این مقدار با `AKfycb...` شروع می‌شود و حدود ۶۰ کاراکتر است. روی **Copy** کنار آن کلیک کنید.

این ID را ذخیره کنید. در گام بعدی به آن نیاز خواهید داشت.

### گام ۴ — ساخت فایل پیکربندی

در پوشه پروژه:

<div dir="ltr">

```bash
cp config.example.json config.json
```

</div>

فایل `config.json` را در یک ویرایشگر متن باز کنید و دو مقداری که تازه ساختید را وارد کنید:

<div dir="ltr">

```json
{
  "mode": "apps_script",
  "google_ip": "216.239.38.120",
  "front_domain": "www.google.com",

  "script_id": "PASTE_YOUR_DEPLOYMENT_ID_HERE",
  "auth_key": "PASTE_YOUR_SHARED_SECRET_HERE",

  "listen_host": "127.0.0.1",
  "listen_port": 8085,
  "socks5_enabled": true,
  "socks5_port": 1080,

  "log_level": "info"
}
```

</div>

حداقل ویرایش‌های لازم:

- `script_id` — Deployment ID از گام ۳.
- `auth_key` — کلید مشترک از گام ۲ (باید با `SHARED_SECRET` در Apps Script شما یکسان باشد).

فایل را ذخیره کنید.

### گام ۵ — اولین اجرا و تولید CA

پراکسی را اجرا کنید:

<div dir="ltr">

```bash
./relayvpn -c config.json
```

</div>

باید خروجی مشابه زیر ببینید:

<div dir="ltr">

```
INFO CA ready cert=/.../ca/ca.crt fingerprint=a1b2c3d4...
INFO HTTP proxy listening addr=127.0.0.1:8085 auth=false
INFO SOCKS5 proxy listening addr=127.0.0.1:1080 auth=false
```

</div>

در اولین اجرا، پراکسی یک Certificate Authority محلی در `ca/ca.crt` و `ca/ca.key` تولید می‌کند. **اثر انگشت SHA-256** چاپ‌شده در زمان راه‌اندازی را یادداشت کنید — برای تایید CA پس از نصب از آن استفاده خواهید کرد.

پراکسی را در این ترمینال در حال اجرا رها کنید. برای گام بعدی یک ترمینال جدید باز کنید.

### گام ۶ — نصب گواهی CA محلی

مرورگر شما باید `ca/ca.crt` را به‌عنوان یک ریشه (root) معتبر بشناسد، وگرنه هر سایت HTTPS خطای گواهی نشان خواهد داد.

> ⚠️ این گواهی **فقط** برای دستگاه شماست. فایل `ca/ca.crt` یا `ca/ca.key` را با هیچ‌کس به اشتراک نگذارید. کلید خصوصی همان چیزی است که به پراکسی اجازه می‌دهد به‌جای هر وب‌سایتی برای مرورگر شما هویت جعل کند؛ هر کسی که این فایل را داشته باشد می‌تواند ترافیک شما را MITM کند.

#### macOS

<div dir="ltr">

```bash
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain ca/ca.crt
```

</div>

یا از طریق GUI: روی `ca/ca.crt` دو بار کلیک کنید، آن را در **Keychain Access** زیر **System** پیدا کنید، دو بار کلیک کنید، **Trust** را گسترش دهید، و گزینه «When using this certificate» را روی **Always Trust** قرار دهید.

#### Linux (Debian/Ubuntu)

<div dir="ltr">

```bash
sudo cp ca/ca.crt /usr/local/share/ca-certificates/relayvpn.crt
sudo update-ca-certificates
```

</div>

#### Linux (Arch/Fedora)

<div dir="ltr">

```bash
sudo cp ca/ca.crt /etc/ca-certificates/trust-source/anchors/relayvpn.crt
sudo update-ca-trust
```

</div>

#### Windows

۱. روی `ca\ca.crt` دو بار کلیک کنید.
۲. روی **Install Certificate** کلیک کنید.
۳. **Current User** را انتخاب و روی **Next** کلیک کنید.
۴. گزینه **Place all certificates in the following store** را انتخاب کنید → **Browse** → گزینه **Trusted Root Certification Authorities** را انتخاب کنید.
۵. روی **Next** → **Finish** کلیک کنید. هشدار امنیتی را تایید کنید.

#### Firefox (همه پلتفرم‌ها — جداگانه لازم است)

Firefox از store گواهی خودش استفاده می‌کند، حتی روی Linux/macOS:

۱. **Settings** → جستجوی «proxy» → **Privacy & Security** → اسکرول به **Certificates** → **View Certificates**.
۲. تب **Authorities** را باز کنید → روی **Import** کلیک کنید.
۳. `ca/ca.crt` را از پوشه پروژه انتخاب کنید.
۴. **Trust this CA to identify websites** را تیک بزنید → روی **OK** کلیک کنید.

#### تایید معتبر بودن CA

پس از نصب، گواهی را در store مرورگر/سیستم‌عامل پیدا کنید و تایید کنید که thumbprint SHA-256 با اثر انگشتی که پراکسی شما در گام ۵ چاپ کرد، یکسان است. اگر یکسان نیستند، **توقف کنید** — مشکلی وجود دارد و باید قبل از ادامه آن را بررسی کنید.

### گام ۷ — تنظیم مرورگر روی پراکسی

#### Firefox

**Settings** → جستجوی «proxy» → **Network Settings** → **Settings...** →

- **Manual proxy configuration** را انتخاب کنید.
- **HTTP Proxy:** `127.0.0.1`، **Port:** `8085`.
- **Also use this proxy for HTTPS** را تیک بزنید.
- روی **OK** کلیک کنید.

#### Chrome / Edge / Brave

این مرورگرها از پراکسی سیستم استفاده می‌کنند. تمیزترین راه نصب یک افزونه proxy switcher مانند **SwitchyOmega** یا **FoxyProxy** و پیکربندی پروفایلی با مقصد `127.0.0.1:8085` است. اینگونه می‌توانید پراکسی را بدون تغییر تنظیمات سراسری سیستم روشن و خاموش کنید.

اگر ترجیح می‌دهید از تنظیمات سیستم استفاده کنید:

- **macOS:** **System Settings** → **Network** → اتصال فعال شما → **Details** → **Proxies** → **Web Proxy (HTTP)** و **Secure Web Proxy (HTTPS)** را فعال کنید، هر دو `127.0.0.1` پورت `8085`.
- **Windows:** **Settings** → **Network & Internet** → **Proxy** → **Manual proxy setup** → **Use a proxy server** = روشن، آدرس `127.0.0.1`، پورت `8085`.

#### تست کنید

<https://www.example.com> را در مرورگر پراکسی‌شده باز کنید. صفحه باید به‌طور عادی بارگذاری شود. اگر شد، کارتان تمام است.

برای تستی محسوس‌تر، یک سایت که می‌دانید از اتصال عادی شما مسدود یا geo-fence شده را باز کنید.

## اشتراک‌گذاری در شبکه محلی (اختیاری)

اگر می‌خواهید سایر دستگاه‌های شبکه شما (موبایل، تبلت، کامپیوتر دوم) از همین پراکسی استفاده کنند، فایل `config.json` را ویرایش کنید:

<div dir="ltr">

```json
{
  "lan_sharing": true,
  "proxy_user": "alice",
  "proxy_pass": "a-second-strong-secret-for-the-proxy-itself"
}
```

</div>

اشتراک‌گذاری شبکه محلی **نیازمند** `proxy_user` و `proxy_pass` است — پراکسی بدون این‌ها از شروع امتناع می‌کند. این کار از سهمیه رله شما در برابر هر کس دیگری در همان شبکه (از جمله wifi کافی‌شاپ، همسایه‌ها، یا دستگاه‌های IoT غیرقابل اعتماد) محافظت می‌کند.

وقتی اشتراک‌گذاری شبکه محلی روشن است:

- پراکسی به‌طور خودکار روی `0.0.0.0` گوش می‌دهد.
- IP شبکه محلی دستگاه خود را پیدا کنید (مثلاً `192.168.1.42`).
- روی سایر دستگاه‌ها، پراکسی را به‌صورت `192.168.1.42:8085` (HTTP) پیکربندی کنید و وقتی پرسیده شد `alice` / رمز عبور خود را وارد کنید.
- باید `ca/ca.crt` را روی هر دستگاهی که از HTTPS از طریق پراکسی استفاده می‌کند، نصب کنید.

## به‌روزرسانی رله

اگر زمانی `apps_script/Code.gs` را تغییر دادید، **ویرایش به‌تنهایی هیچ کاری نمی‌کند** — Apps Script نسخه قبلاً مستقرشده را همچنان سرو می‌کند. باید:

۱. در ویرایشگر Apps Script روی **Deploy** → **New deployment** کلیک کنید.
۲. پیکربندی کنید (Web app، Execute as Me، Anyone access).
۳. Deployment ID **جدید** را کپی کنید.
۴. مقدار `script_id` را در `config.json` جایگزین کنید.
۵. `relayvpn` را راه‌اندازی مجدد کنید.

## عیب‌یابی

<div dir="ltr">

| Problem | Fix |
|---|---|
| `config: auth_key must be ≥24 chars` | کلید مشترک شما خیلی کوتاه است. با `openssl rand -base64 32` یک مقدار جدید بسازید. |
| `config: auth_key is still the example placeholder` | فراموش کرده‌اید `CHANGE_ME_TO_A_STRONG_SECRET` در `config.json` را جایگزین کنید. |
| Browser shows "Your connection is not private" on every site | گواهی CA به‌درستی نصب نشده است. گام ۶ را برای سیستم‌عامل خود دوباره انجام دهید، مرورگر را به‌طور کامل ببندید (همه پنجره‌ها، از جمله نمادهای system tray). |
| Worked, but stopped after `git pull` | احتمالاً CA دوباره تولید شده است. بررسی کنید اثر انگشت نمایش‌داده‌شده در راه‌اندازی با مقدار موجود در trust store یکسان باشد. اگر نه، root قدیمی را پاک کرده و جدید را نصب کنید. |
| `relay error: bad sig` | مقدار `auth_key` در `config.json` با `SHARED_SECRET` در Apps Script شما یکسان نیست. باید بایت‌به‌بایت برابر باشند. |
| `relay error: stale` | ساعت کامپیوتر شما بیش از ۶۰ ثانیه با زمان واقعی اختلاف دارد. NTP / تنظیم خودکار زمان را در سیستم‌عامل خود فعال کنید. |
| `relay HTTP 502` from the proxy | Apps Script یک خطا برگردانده. محتمل‌ترین دلیل: deployment پس از ویرایش `Code.gs` دوباره ساخته نشده، یا به سهمیه روزانه رسیده‌اید. یک deployment جدید بسازید و `script_id` را به‌روز کنید. |
| Proxy works for a while then fails | احتمالاً به سهمیه ۲۰٬۰۰۰ درخواست در روز Apps Script رسیده‌اید. تا نیمه‌شب وقت اقیانوس آرام صبر کنید، یا رله‌های بیشتری روی حساب‌های گوگل اضافی مستقر کنید و همه را در `script_ids` لیست کنید. |
| Telegram works on HTTP but not SOCKS5 | انتظار می‌رود. کلاینت‌های SOCKS5 IP خام بدون hostname می‌فرستند که قابل رله نیست. تلگرام را به‌جای آن به‌عنوان پراکسی HTTP روی `127.0.0.1:8085` پیکربندی کنید. |
| `lan_sharing=true requires proxy_user and proxy_pass` | اشتراک‌گذاری شبکه محلی بدون احراز هویت، رله شما را در معرض هر کسی در شبکه شما قرار می‌دهد. هر دو فیلد را تنظیم کنید. این محدودیت عمدی است. |
| First HTTPS site to a new domain is slow | هر hostname جدید نیازمند یک گواهی leaf تازه است. سایت‌های بعدی زیر همان دامنه والد، آن را دوباره استفاده می‌کنند. |

</div>

برای لاگ‌های تفصیلی، با `"log_level": "debug"` در `config.json` اجرا کنید.

## نکات امنیتی

- **با `config.json` به‌عنوان یک راز رفتار کنید.** این فایل کلید مشترک رله را دارد. اگر فاش شود، یک حمله‌کننده می‌تواند تا زمانی که با کلید جدید دوباره مستقر کنید، از سهمیه گوگل شما استفاده کند.
- **با `ca/ca.key` به‌عنوان یک راز رفتار کنید.** هر کسی این فایل را داشته باشد می‌تواند گواهی‌ای بسازد که هویت هر وب‌سایتی را برای هر کسی که به CA شما اعتماد می‌کند، جعل کند — که فقط شمایید، اما با این حال: آن را به اشتراک نگذارید، در git قرار ندهید، و به cloud drive‌ها همگام‌سازی نکنید.
- **CA را روی کمترین دستگاه ممکن نصب کنید.** هر دستگاهی که به CA شما اعتماد دارد، دستگاهی است که می‌تواند توسط `ca/ca.key` MITM شود. اگر برای موبایل اشتراک‌گذاری شبکه محلی راه‌اندازی می‌کنید، CA را فقط روی آن موبایل نصب کنید، نه کل خانه.
- **`validate_https` را در هیچ‌جا غیرفعال نکنید.** پای فرانت‌شده TLS همیشه گواهی `front_domain` را اعتبارسنجی می‌کند. غیرفعال‌سازی آن، ترافیک شما را در معرض MITM توسط هر کسی بین شما و گوگل قرار می‌دهد.
- **`auth_key` را به‌طور دوره‌ای rotate کنید.** یک کلید جدید بسازید، در `config.json` و `SHARED_SECRET` پیست کنید، Apps Script را دوباره deploy کنید.
- **اثر انگشت CA را در هر راه‌اندازی تازه بررسی کنید.** پراکسی آن را در زمان شروع چاپ می‌کند؛ با مقدار موجود در store گواهی سیستم‌عامل/مرورگر خود مقایسه کنید.

## سلب مسئولیت

**این نرم‌افزار فقط و فقط برای اهداف آموزشی، پژوهشی، و تست شخصی ارائه شده است.**

با استفاده از این ابزار شما موافقت می‌کنید که:
